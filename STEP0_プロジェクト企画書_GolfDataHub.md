# STEP 0: プロジェクト企画書（確定版）

## プロジェクト名（仮）

**Golf Data Link (GDL)** - ゴルフスコア・データハブ構想

## 1. 企画概要 (Executive Summary)

紙のゴルフスコアカードをOCRで読み取り、デジタル資産として保存する**iOS / Android 対応モバイルアプリ**。
最大の特徴は、自社アプリ内での分析に留まらず、**「データの標準化と外部配信」**を主眼に置いている点である。ユーザーのスコアデータを「特定のアプリ」から解放し、あらゆるゴルフサービスで活用可能な「ポータブルな資産」に変える。

## 2. コア機能要件 (Core Features)

### A. 高精度OCRスキャン (Input)

* **対象:** 紙のゴルフスコアカード。
* **取得データ（画像解析）:**
  * ホール毎のスコア
  * パット数 (Putts)
  * 同伴者のスコア
  * 同伴者のパット数
* **技術要件:** 撮影環境（ゴルフ場のカート内、逆光など）に左右されない高い認識精度（※STEP 1で設計した前処理技術がここで活きる）。

### B. パーソナル・スコア・データベース (Storage)

* ユーザーごとのクラウドデータベース構築。
* **保存データ項目（メタデータ拡充）:**
  * **ゴルフ場情報:**
    * **ゴルフ場名**
    * **コース名**（OUT/IN、東/西コースなど）
    * **ティー情報**（カラー：Red/White/Blue/Black、または距離レート）
    * ※スコアカードから読み取れない場合は、**GPS位置情報による自動提案**やユーザー選択で補完する。
  * **プレイ環境:**
    * 日付
    * 天気・気温（日時と場所からAPIで自動取得）

### C. データ標準化と配信 (Output / The "Hub")

* **標準フォーマット策定:**
  * 他のアプリが取り込みやすい汎用形式（JSON / XML / CSV）を定義。
  * （例: `GolfScoreXML 1.0` 規格の策定）
* **データエクスポート/連携:**
  * API連携、またはファイルエクスポート機能により、他社のゴルフ管理アプリ、分析ツール、SNS等へデータを「流す」機能。

## 3. プラスアルファの価値提案 (Value Proposition +α)

企画の価値を最大化するための追加アイデア案：

### アイデア1: 「スコアの原本証明書」 (Trust)

* **概要:** 手入力されたスコア（自己申告）は改ざんが容易だが、紙のカードの画像とセットで保存されたデータは「証拠能力」が高い。
* **価値:** コンペやハンディキャップ取得時の「公式記録」としての信頼性を担保する機能。「Verified by GDL」マークの付与。

### アイデア2: 「過去の思い出のデジタル化」 (Legacy)

* **概要:** ラウンド直後だけでなく、ユーザーの家の引き出しに眠っている「過去10年分のスコアカード」をまとめてスキャンし、生涯データを構築する機能。
* **価値:** 過去のデータが蘇ることで、アプリへの定着率（ロックイン）が劇的に高まる。

### アイデア3: 「同伴者へのデータAirDrop / Nearby Share」 (Social)

* **概要:** 代表者1名がカードをスキャンすれば、標準化されたデータ形式でその場にいる同伴者へ即座に共有・配信できる。
* **価値:** 「入力係」の負担が報われる機能。OSを問わずシェアできる仕組みで、アプリのバイラル係数（口コミ拡散）を高める。

## 4. マネタイズ戦略 (Monetization)

「データを囲い込まない」という特性を活かしたビジネスモデルを提案。

### A. Freemium (B2C)

* **Free:**
  * OCR読み取り（月5枚まで）
  * 基本データ保存
  * 標準フォーマットでのエクスポート（広告あり）
* **Pro (サブスクリプション / 月額500円想定):**
  * 無制限OCR読み取り（過去分の一括取込に最適）
  * **APIアクセス権:** マイデータをIFTTTやZapier、Googleスプレッドシート等に自動連携する機能。
  * 同伴者データの出力。

### B. データ・アライアンス (B2B)

* **ゴルフ場/予約サイトへの送客:**
  * スコアデータから「相性の良いゴルフ場」を分析し、予約サイト（提携先）へ誘導するアフィリエイトモデル。
* **OEM提供:**
  * この高精度OCRエンジン自体を、APIとして他のゴルフアプリ事業者に提供する（従量課金）。

## 5. 技術スタック (再確認)

* **OS:** iOS / Android (Cross-Platform Framework: React Native or Flutter)
* **OCR:** On-device processing (ML Kit) for privacy & speed.
* **Backend:** Firebase or AWS (User DB & API Gateway).
